    USERS {

        bigint id PK

        string name

        string email UK

        timestamp email_verified_at

        string password

        int status "1:Active, 0:Inactive"

        timestamp last_login_at

    }



    CLUBS {

        bigint id PK

        string name

        string unique_id UK

        string email

        text description

        string logo

    }



    ASSESSMENTS {

        bigint id PK

        bigint club_id FK

        string title

        int maturity_level

        json report_data

        timestamp reporting_date

        timestamp grace_period_expires_at

    }



    SUBSCRIPTIONS {

        bigint id PK

        bigint club_id FK

        string plan_type "Elite, Basic"

        string status "Active, Expired"

        decimal amount

        date end_date

    }



    AUDIT_LOGS {

        bigint id PK

        bigint user_id FK

        string action

        text description

        string ip_address

    }


2. Data Dictionary (Core Tables)



2.1 Table: `users`


ColumnTypeNullableDescription

`id`BIGINTNoPrimary Key
`name`VARCHAR(255)NoFull legal name of the operative
`email`VARCHAR(255)NoUnique identifier and communication node
`status`TINYINTNoBinary operational state (1=Active, 0=Suspended)

2.2 Table: `clubs`


ColumnTypeNullableDescription

`id`BIGINTNoPrimary Key
`unique_id`VARCHAR(255)NoInstitutional identifier (e.g., GC-4421)
`name`VARCHAR(255)NoOfficial club designation

2.3 Table: `assessments`


ColumnTypeNullableDescription

`id`BIGINTNoPrimary Key
`maturity_level`INTYesDerived score from 1 (Seeding) to 5 (Soaring)
`report_data`JSONYesEncapsulated diagnostic outcomes
