Growing Cents: Test Data Reset Procedure



To maintain operational purity during staging, follow this protocol to reset all nodes to a verified baseline.

Phase 1: Institutional Purge



1. Clear Assessments: Delete all records from `assessments` and `recommendations` tables.
2. Reset Clubs: Remove all non-baseline institutional entries.
3. Wipe Logs: Truncate the `audit_logs` table (preserve 'critical' tags if needed).

Phase 2: User Synchronization



1. De-register Non-Admins: Clear the `users` table of all but the primary Command nodes.
2. Clear 2FA Tokens: Expire all pending verification tokens.
3. Reset Password History: Refresh salt and pepper configs if staging is compromised.

Phase 3: Financial Baseline



1. Expire Subscriptions: Set all active test subscriptions to `expired`.
2. Clear Mock Webhooks: Flush the `webhook_events` cache.


> [!WARNING]
> This procedure is IRREVERSIBLE. Ensure a High Command backup exists before execution.
