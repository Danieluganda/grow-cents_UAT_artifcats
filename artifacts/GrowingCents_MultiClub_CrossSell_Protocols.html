Multi-Club & Cross-Selling Protocols

This document defines how the platform handles users who belong to multiple investment clubs, how data is isolated between clubs, how subscription status is managed per club, and the current scope of cross-selling logic as defined in the SRS.

1. Multi-Club Membership Rules

A single user account can be associated with multiple investment clubs. The relationship is managed through the club_user pivot table:

ColumnValuesDescription
user_idFK → usersThe user
club_idFK → clubsThe club
role'admin' or 'member'The user's role within that specific club

A user can hold different roles in different clubs simultaneously — e.g. Admin in Kampala Investment Club and Member in Entebbe Savings Club.

2. Club Selection Logic After Login

After successful 2FA verification, TwoFactorController@store() routes the user based on club count:

ConditionRouteBehaviour
User has Super Admin Spatie roleadmin.dashboardBypasses club routing entirely — sees all clubs
User belongs to exactly 1 clubclubs.dashboard/{club_id}Auto-directed — no selector shown
User belongs to 2+ clubsclubs.selectClub selector page — dropdown or tiles showing user's clubs
User belongs to 0 clubsdashboardGeneric dashboard — no club context

3. Data Isolation Across Clubs

Club data is strictly isolated — a user in Club A cannot access Club B's assessments or reports, even if they have the same Spatie system role. Isolation is enforced in every controller via:

// AssessmentController — all queries scoped to the authenticated club
$club = Club::findOrFail($clubId);
$assessment = Assessment::where('club_id', $clubId)->findOrFail($assessmentId);

If a user guesses another club's URL, findOrFail() returns 404 — not a permission error — preventing information disclosure.

4. Subscription Isolation Across Clubs

Each club has its own subscription record in the subscriptions table:

  • A paid subscription for Kampala Investment Club does not unlock reports for Entebbe Savings Club
  • Each new assessment submission triggers a new billing event for that specific club
  • Subscription status is checked per club: $club->currentSubscription()
  • Free access: complete assessment + view overall score. Paid access: full report, PDF, email, past assessments.
📌 UAT Note: Payment gate is currently inactive — all reports are accessible without subscription during UAT. Subscription gating will be activated once Pesapal merchant account is agreed.

5. Cross-Selling & Report Unlocking Logic

Per SRS Section 9, the conditions for report access are:

FeatureFree (No Subscription)Paid Subscription
Complete Assessment✅ Yes✅ Yes
View Overall Score✅ Yes✅ Yes
View Full Report / Dashboard❌ No✅ Yes
Download PDF❌ No✅ Yes
Email Report❌ No✅ Yes
Access Past Assessments❌ No✅ Paid ones only
Growing Cents eBook Link✅ Link only✅ Linked in report

The upsell trigger is the report paywall — after submitting an assessment, free users see their overall score but are prompted to subscribe to unlock the full report. This is implemented in AssessmentController@show() via $club->currentSubscription().
