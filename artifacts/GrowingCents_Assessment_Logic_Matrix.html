Assessment Logic Matrix

Technical specification for the scoring algorithm, maturity thresholds, conditional logic rules, and assessment lifecycle as implemented in the platform. All scoring logic mirrors the Excel-based tool (vFINAL) provided by Growing Cents Limited.

2. Scoring Architecture — 3 Dimensions

Every response option in the JSON schema (storage/app/data/complete_assessment_data.json) carries three weight dimensions:

Weight DimensionJSON KeyUsed For
Response Score for Statementoverall_scoreContributes to overall club score
Response Score for Areaarea_scoreContributes to the area percentage
Response Score for Sectionsection_scoreContributes to the section percentage

For each dimension, the engine accumulates actual_score and max_possible_score then computes percentage = (actual / max) * 100.

4. Conditional Logic Rules

The platform implements 384 conditional rules loaded from storage/app/data/conditional_rules.json. These rules dynamically show, hide, or make mandatory specific questions based on earlier responses — mirroring the if/else branches in the Excel tool.

How Conditional Logic Works

  • Rules are passed to the frontend as a JavaScript object on assessment form load
  • When a user changes a response, the frontend JS evaluates all dependent rules and shows/hides affected questions in real time
  • Hidden questions are not stored in the responses JSON on submission — they score 0
  • Not Applicable (option value = 6) responses carry weight = 0 in all three dimensions

Example Rule Structure

{
  "trigger_question_id": "1Structure&Naturei",
  "trigger_value": 6,
  "affected_questions": ["1Structure&Natureii", "1Structure&Natureiii"],
  "action": "hide"
}

6. Assessment Edit Rules

  • Responses cannot be edited after submission (status = Completed)
  • SRS Section 9 defines a 7-day grace period — automated. Implementation: grace_period_expires_at field on assessments table, set to now() + 7 days on submission
  • After grace period expires: immutable. No edit endpoint accessible.
  • Assessment date (reporting_date) is set from KYC question 0aii and cannot be changed post-submission
  • Club name and inception date are uneditable in Area 0 KYC on repeat assessments (enforced at frontend)
