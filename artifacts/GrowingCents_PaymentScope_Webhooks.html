Payment Scope & Webhooks

This document defines the payment integration scope for the Growing Cents platform as agreed in the signed SRS, the current UAT status of payment features, the subscription billing model, and the webhook payload structure for when Pesapal integration is activated.

📌 Current Status: Payment integration is explicitly deferred in SRS Section 12 — "Subscription Gateway — Pesapal — pending agreement on merchant account." The payment gate is currently inactive (commented out). All reports are accessible during UAT without payment. Pesapal integration will be built and tested as a separate phase once the merchant account is active.

1. Subscription Model (Per SRS Section 9)

FeatureFree (No Subscription)Paid Subscription

  • Each club has a unique subscription account
  • Each new assessment submission triggers a new billing event
  • Completed and paid-for reports remain accessible permanently to that club

2. Billing Rules

  • Billing unit: per assessment — each assessment submission creates a new billing event for that club
  • Billing is per club, not per user — one subscription covers all club members' access to that assessment's report
  • Subscription plan tiers are to be defined by Growing Cents Limited — not yet finalised in SRS
  • SubscriptionID syntax (per SRS Section 5): must include sequential numbering + paymentRef. Format: GCS-{YEAR}-{0001}-{PAYREF}

3. Payment Lifecycle States

4. Mock Payment Scope for UAT

Since Pesapal is not yet integrated, the following mock approach is used during UAT:

  • Payment gate is commented out in AssessmentController@show() and @downloadPdf() — all reports accessible
  • The subscriptions table exists and the model is complete — data can be manually inserted for testing subscription logic
  • The SubscriptionController has plans(), store(), and callback() routes scaffolded for Pesapal
  • Moses can test subscription-related UAT scenarios (G1–G2) by directly seeding a subscription record

5. Pesapal Webhook Payload Structure

When Pesapal integration is activated, the callback endpoint GET /clubs/{club}/subscribe/callback will receive the following from Pesapal:

// Pesapal IPN (Instant Payment Notification) — query parameters
GET /clubs/1/subscribe/callback
  ?OrderTrackingId=PESAPAL-TXN-20260315-001
  &OrderMerchantReference=GCS-2026-0001-PESAPAL
  &OrderNotificationType=IPNCHANGE

Expected Webhook Handling Flow

// SubscriptionController@callback()
// 1. Receive Pesapal IPN parameters
// 2. Call Pesapal API to verify transaction status
// 3. If status === 'COMPLETED':
Subscription::where('payment_reference', $orderTrackingId)
    ->update([
        'status'     => 'active',
        'start_date' => now(),
        'end_date'   => now()->addYear(),
    ]);
// 4. AuditLog::log('subscription_activated', clubId)
// 5. Redirect to assessment report
