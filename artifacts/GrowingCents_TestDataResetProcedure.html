Test Data Reset Procedure

Step-by-step procedure for resetting the Growing Cents staging environment between UAT test runs. This procedure restores all test accounts, clubs, and assessments to a known baseline state so Moses can repeat tests from a clean starting point.

⚠️ Critical: Always verify APP_ENV is staging — NEVER run these commands on the production server at assessment.growing-cents.com.

1. Verify Environment Before Reset

# SSH into the staging server
ssh user@staging-server

# Navigate to project root
cd /var/www/growing-cents

# Confirm you are on staging — must output 'staging'
php artisan env

# Confirm you are on the uat/staging branch
git branch --show-current
# Expected output: uat/staging

2. Option A — Soft Reset (Recommended for Between Tests)

Wipes assessments, subscriptions, and audit logs. Keeps user accounts and clubs intact. Use this between individual test sections (e.g. after completing Section C, before starting Section D).

# Run the UAT soft reset seeder
php artisan db:seed --class=UATSoftResetSeeder

# What this does:
# 1. Truncates: assessments, assessment_responses, subscriptions, audit_logs
# 2. Reseeds: 2 clubs (Kampala IC + Entebbe SC) with 5 assessments at varied maturity levels
# 3. Resets: all user passwords to original UAT values
# 4. Preserves: user accounts, roles, permissions, clubs

3. Option B — Full Reset (Nuclear — Use With Caution)

Complete wipe and reseed. Destroys ALL data including users, roles, clubs. Use only at the start of a new UAT cycle or if the DB is in an unrecoverable state.

# Full wipe and reseed
php artisan migrate:fresh --seed

# What this does:
# 1. Drops ALL tables
# 2. Re-runs ALL migrations
# 3. Runs DatabaseSeeder (which calls UATSeeder)
# 4. Re-creates: all roles, permissions, test accounts, clubs, assessments

# After migrate:fresh, regenerate app key if needed
php artisan key:generate

# Clear all caches
php artisan config:clear
php artisan cache:clear
php artisan view:clear
php artisan route:clear

6. Clear OTP Tokens After Reset

If any test account is stuck on the 2FA screen (OTP expired but not cleared), run:

# Clear all pending OTP tokens — forces fresh OTP on next login
php artisan tinker
>>> App/Models/User::query()->update(['otp_code' => null, 'otp_expires_at' => null]);
>>> exit

7. Reset Mailtrap Inbox

After a full reset, clear the Mailtrap inbox so Moses starts with a clean email log:

  1. Log into mailtrap.io
  2. Open the Growing Cents UAT inbox
  3. Click Clear Inbox (top right button)
  4. All previous OTP emails, invitation emails, and report emails are cleared

8. Verify Reset Was Successful

# Check assessment count
php artisan tinker
>>> App/Models/Assessment::count();  // Expected: 5
>>> App/Models/User::count();        // Expected: 5 (UAT accounts)
>>> App/Models/Club::count();        // Expected: 2
>>> exit

# Check roles seeded correctly
php artisan permission:show

9. If Something Goes Wrong

  • Migration fails: Check storage/logs/laravel.log for the error. Most common cause: duplicate column or table already exists. Run php artisan migrate:status to see which migrations ran.
  • Seeder fails: Run seeder in isolation: php artisan db:seed --class=UATSeeder. Check for unique constraint violations — may mean the data already exists from a partial run.
  • Permission errors: Re-run php artisan permission:cache-reset then try again.
  • App key error after fresh: Run php artisan key:generate
