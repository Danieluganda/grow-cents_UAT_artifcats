API Sequence Diagrams

This document covers the 5 core interaction flows in the Growing Cents platform — traced through each layer from browser request to database response. All actors, route names, controller methods, and response types reflect the actual codebase.

1. User Login + 2-Step Verification (2FA)

Browser          Router           Middleware       Controller       Model/DB         Mail
  |                |                |                |                |                |
  |-- POST /login (email,pass) ---> |                |                |                |
  |                |-- throttle:login check -------> |                |                |
  |                |                |-- AuthenticatedSessionController@store           |
  |                |                |                |-- User::where('email') -------> |
  |                |                |                | <------------- User record ----- |
  |                |                |                |-- Hash::check(pass, hash)       |
  |                |                |                |-- Generate OTP rand(100000,9..) |
  |                |                |                |-- user->update(otp_code, expires_at)
  |                |                |                |-- Mail::send(TwoFactorCode) --> |
  |                |                |                |                | <-- sent OK --- |
  | <--- redirect GET /verify-2fa --|                |                |                |
  |                |                |                |                |                |
  |-- POST /verify-2fa (code) ----> |                |                |                |
  |                |-- auth middleware ------------> |                |                |
  |                |                |-- TwoFactorController@store     |                |
  |                |                |                |-- check code===otp_code AND not expired
  |                |                |                |-- user->update(otp_code=null, last_login_at=now())
  |                |                |                |-- session->put('2fa_verified', true)
  |                |                |                |-- AuditLog::log('login_success')
  |                |                |                |-- hasRole('Super Admin')? -> admin.dashboard
  |                |                |                |-- clubs()->count()==1? -> clubs.dashboard/{id}
  |                |                |                |-- clubs()->count()>1? -> clubs.select
  | <--- redirect to destination ---|                |                |                |

ERROR PATHS:
  Invalid password -> 422 {"email": ["These credentials do not match our records."]}
  Wrong OTP        -> 422 {"code": ["The verification code is invalid or has expired."]}
  Expired OTP      -> 422 (otp_expires_at < now())
  Too many resends -> 429 (throttle:3,1 on /verify-2fa/resend)

5. Multi-Club Login Routing

After successful 2FA in TwoFactorController@store():

user->hasRole('Super Admin')?
  YES -> redirect admin.dashboard (bypasses all club routing)
  NO  -> user->clubs()->count()

  count == 0 -> redirect /dashboard (no club assigned)
  count == 1 -> $club = user->clubs()->first()
                redirect clubs.dashboard/{club->id}  [auto-direct, no selector]
  count > 1  -> redirect clubs.select

// clubs.select page:
GET /clubs/select -> ClubController@select
  $clubs = auth()->user()->clubs  [scoped to user's pivot records only]
  render clubs/select view

// User picks a club:
GET /clubs/{club}/dashboard -> DashboardController@club
  Club::findOrFail($clubId)  [validates user belongs to this club]
  load club-scoped data
  render club dashboard
