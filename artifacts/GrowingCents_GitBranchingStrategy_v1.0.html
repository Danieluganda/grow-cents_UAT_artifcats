Git Branching Strategy v1.0

This document defines the Git branching model for the Growing Cents platform repository. It covers branch purposes, merge rules, Moses's access scope during UAT, and how to handle hotfixes during the UAT period.

1. Branch Structure

BranchPurposeWho DeploysDirect Push?
mainProduction — live code at assessment.growing-cents.com. Stable, tested, signed-off releases only.Daniel (CTO)No — merge from develop only after UAT sign-off
developActive development branch. All feature work merges here first. This is Daniel's working branch during UAT remediation.DanielYes — Daniel works here
uat/stagingUAT snapshot branch. Branched from develop at the start of UAT. Frozen — only bug fixes from UAT findings are merged here. Moses has read-only access to this branch.DanielNo — only hotfixes merged in
hotfix/*Emergency fixes during UAT. Branched from uat/staging, fixed, merged back to both uat/staging and develop.DanielYes — short-lived

2. Branch Lifecycle During UAT

How UAT/Staging Was Created

git checkout develop
git checkout -b uat/staging
git push origin uat/staging
# Moses invited as read-only collaborator on this branch

Flow When a UAT Bug Is Found

# 1. Moses logs a bug in the UAT workplan
# 2. Daniel creates a hotfix branch from uat/staging
git checkout uat/staging
git checkout -b hotfix/fix-submit-permission

# 3. Daniel fixes the bug (e.g. restores submit_assessment middleware)
# 4. Merge hotfix into BOTH uat/staging AND develop
git checkout uat/staging
git merge hotfix/fix-submit-permission
git push origin uat/staging

git checkout develop
git merge hotfix/fix-submit-permission
git push origin develop

# 5. Delete hotfix branch
git branch -d hotfix/fix-submit-permission

3. Moses's Access — Read Only

What Moses Can DoWhat Moses Cannot Do
Clone or pull the uat/staging branchPush to any branch
Browse commit history on uat/stagingMerge or delete branches
View file contents to verify code matches documentationCreate branches
Verify fixes after Daniel deploys a hotfixAccess main or develop

Clone Command for Moses

git clone --branch uat/staging --single-branch https://github.com/[repo-url].git growing-cents-uat

4. Merge Rules

  • develop → uat/staging: Only for UAT-confirmed fixes. Requires Daniel's review.
  • uat/staging → main: Only after full UAT sign-off by Moses and Emmanuel.
  • hotfix/* → both uat/staging and develop: Required for every hotfix — never merge to only one.
  • Never merge directly to main without a signed UAT completion document.

5. Commit Message Convention

All commits during UAT remediation follow this format so Moses can trace fixes to specific UAT test IDs:

fix(F3.1): restore submit_assessment middleware to AssessmentController
fix(C5.6): add 7-day grace period field to assessments table
fix(B1.2): add country dropdown to club registration form
fix(L6): configure UptimeRobot monitoring
feat(H2): add scheduled nudge command for idle assessments

6. Environment Variables

The .env file is never committed to Git — it is in .gitignore. The staging environment is configured separately. If Moses needs to verify environment configuration, Daniel will share a sanitised .env.example showing all required keys without values.
