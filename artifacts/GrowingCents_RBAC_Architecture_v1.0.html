RBAC Architecture v1.0

This document covers the full 6-layer implementation of Role-Based Access Control in the Growing Cents platform — from database schema through to view rendering. Built on Laravel 11 and Spatie Laravel-Permission v6.

For the conceptual 7-layer interaction model (Resource → Action → Permission → Role → User → Policy), see RBAC Model v2.0.

Layer 3 — Authentication & 2FA Flow

Every session starts here. 8 steps from credential validation to role-based routing:

  1. User submits email + password → AuthenticatedSessionController
  2. Credentials validated against bcrypt hash in users table
  3. OTP generated → stored as otp_code, expires otp_expires_at = now()+10min
  4. OTP emailed via TwoFactorCode mailable
  5. User submits OTP → TwoFactorController@store() validates: code matches AND not expired
  6. OTP cleared: otp_code = null, otp_expires_at = null — cannot be reused
  7. Session flag set: session->put('2fa_verified', true)
  8. Routed by role: Super Admin → admin.dashboard | multi-club → clubs.select | single club → clubs.dashboard/{id}

Controller-Level Declaration (HasMiddleware)

class AssessmentController extends Controller implements HasMiddleware {
    public static function middleware(): array {
        return [
            new Middleware('permission:view_reports',  only: ['index', 'show']),
            new Middleware('permission:export_data',   only: ['downloadPdf', 'export']),
            // NOTE: submit_assessment middleware was removed — being restored (KI-001)
        ];
    }
}

Layer 5 — Controller-Level Checks

Club-Scoped Data Isolation

// Prevents horizontal access — Club A cannot read Club B's data
$club       = Club::findOrFail($clubId);
$assessment = Assessment::where('club_id', $clubId)->findOrFail($id);

Super Admin Bypass

$query = $user->hasRole('Super Admin')
    ? Assessment::with('club')              // sees ALL clubs
    : $club->assessments()->with('club');   // sees own club only

Layer 6 — View-Level Conditional Rendering

{{-- Submit button — Club Admin and Super Admin only --}}
@can('submit_assessment')
    <button type="submit">Submit Assessment</button>
@endcan

{{-- Admin panel link — Super Admin only --}}
@role('Super Admin')
    <a href="{{ route('admin.dashboard') }}">Admin Panel</a>
@endrole

{{-- PDF download — export_data permission required --}}
@can('export_data')
    <a href="{{ route('assessments.download', [$club, $assessment]) }}">Download PDF</a>
@endcan
ℹ️ Important: View-layer checks are a UX convenience — they hide buttons the user cannot use. The real security boundary is always Layer 4 (middleware) and Layer 5 (controller). A user bypassing the UI and hitting the endpoint directly is still blocked at the middleware level.
